# Dev Panel

The dev panel is the runtime inspection surface for strategy development.

## What It Shows Today

The current panel can show:

- turn state
- worker status
- token breakdown
- prompt messages
- tool events
- console output
- context snapshots
- strategy config values

These values come from host and worker dev events emitted during context build and turn end.

## Why It Matters

The dev panel is the easiest way to verify:

- what prompt was actually built
- how much token budget was used
- what tools were exposed
- whether a worker fallback or error occurred

## Limitations

The dev panel is a debugging surface, not the source of truth for the actual send pipeline.

The real execution path still runs through the same host streaming and persistence flow used by normal conversations.
