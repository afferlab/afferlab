#

## Shape

```ts
interface StateTools {
  get<T = unknown>(key: string): Promise<T | null>
  set(key: string, value: unknown): Promise<void>
  delete(key: string): Promise<void>
  has(key: string): Promise<boolean>
}
```

## Overview

`ctx.tools.state` provides persistent key-value storage that strategies can use to store and retrieve state.

This storage is typically used for **conversation-scoped data**, allowing strategies to maintain information across turns.

Examples of stored state include:

- conversation progress
- temporary variables
- intermediate results
- cached values

State is persisted by the runtime and remains available across multiple turns of the same conversation.

### get(key)

Retrieves a value from the state store.

**key:** `string`
The key used to retrieve the stored value.

**Return:** `Promise<T | null>`
Returns the stored value associated with the key.
If the key does not exist, `null` is returned.

Example:

```
const step = await ctx.tools.state.get<number>("step")
```

### set(key, value)

Stores a value in the state store.

**key:** `string`
The key used to store the value.

**value:** `unknown`
The value to store. Values should be JSON-serializable.

The value is persisted by the runtime and can be retrieved in later turns.

Example:

```
await ctx.tools.state.set("step", 3)
```

### delete(key)

Removes a value from the state store.

**key:** `string`
The key to delete.

If the key exists, the value is removed from the store.
If the key does not exist, the operation has no effect.

Example:

```
await ctx.tools.state.delete("step")
```

### has(key)

Checks whether a key exists in the state store.

**key:** `string`
The key to check.

**Return:** `Promise<boolean>`
Returns `true` if the key exists, otherwise `false`.

Example:

```
if (await ctx.tools.state.has("summary")) {
  const summary = await ctx.tools.state.get("summary")
}
```

## Usage

Example: tracking conversation progress.

```ts
export async function onTurnEnd(ctx) {

  const step = (await ctx.tools.state.get("step")) ?? 0

  await ctx.tools.state.set("step", step + 1)
}
```

Example: storing intermediate results.

```
await ctx.tools.state.set("draft_summary", summary)
```

## Notes

State values persist across turns within the same conversation.

Values stored in the state store should be JSON-serializable so they can be safely persisted and restored by the runtime.

The runtime implementation (such as SQLite) is abstracted away from strategies.
